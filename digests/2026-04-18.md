## 官方博客

**Anthropic Engineering** — Scaling Managed Agents: Decoupling the brain from the hands

Anthropic 发布了 Managed Agents 架构设计详解，核心思想是将 agent 的各个组件解耦：session（持久化的日志）、harness（调用 Claude 并路由工具调用的循环）和 sandbox（执行代码的环境）。每个组件都可以独立失败或替换，互不干扰。

关键设计包括：用 execute() 接口让 harness 从容器中分离出来，使容器变成可替换的"cattle"而非需要维护的"pet"；session 不在 Claude 的 context window 内部，而是通过 getEvents() 接口让 brain 自行查询，实现可恢复的上下文存储；凭证不放在 sandbox 里，而是通过 vault 和 MCP proxy 来安全管理，避免 prompt injection 攻击。

这个架构让 p50 time-to-first-token 下降约 60%，p95 下降超过 90%。

https://www.anthropic.com/engineering/managed-agents

## 播客

**Latent Space** — Notion's Token Town: 5 Rebuilds, 100+ Tools, MCP vs CLIs and the Software Factory Future — Simon Last & Sarah Sachs of Notion

Notion 的 AI 团队负责人 Simon Last 和 Sarah Sachs 分享了 Notion 在 AI agent 方面的深度探索。他们从 2022 年底开始尝试构建 AI assistant，历经多次重建：最早的 JavaScript 运行时方案因模型能力不足而失败；后来改用 XML 格式的工具调用，但发现模型不擅长遵循自定义格式；关键转折是将 Notion 的数据模型转化为模型更熟悉的 SQLite 查询，使 Everything is a database 真正为 agent 所用。

关于 MCP vs CLI，Simon 认为 CLI 的最大优势是自举能力——agent 可以直接调试和修复自己，而 MCP 出问题时 agent 无能为力。但 Sarah 补充说，Notion 会同时支持两种方式，因为不同工具有不同的质量需求。

Notion 的 custom agent 采用了"flippy"设计——编辑和对话使用同一个界面，用户可以直接和 agent 聊天来修改它的设置，这种对话式配置大幅降低了使用门槛。

关于 pricing，Notion 采用了 credits 抽象来应对不同模型和工具的差异化定价，同时设计了"auto model picker"来引导用户根据任务选择合适的模型，而不是默认用最贵的 Opus。

Simon 认为 AI 时代的软件工程正在发生变化：人类从写代码转向观察和维护外层系统，模型会驱动完整的开发-调试-验证-部署流程。

Sarah 提出了一个重要观点：Notion 的核心竞争力不在于 MCP 协议或某个特定模型，而在于对"人们如何协作完成工作"这件事的深刻理解——这和 Datadog 与 AWS 的关系类似，Datadog 之所以能存在是因为云存在，但 Datadog 的专长是理解用户需要什么样的可观测性。
