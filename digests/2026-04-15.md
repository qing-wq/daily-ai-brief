## 官方博客

**Claude Blog**
[Redesigning Claude Code on desktop for parallel agents](https://claude.com/blog/claude-code-desktop-redesign)

很多人对 agentic 编程的印象还停留在"一个 prompt 出去，等结果回来"的模式。但现实已经变了：你同时在推进一个代码库的重构、一个 Bug 修复、一个测试用例编写，在多个窗口之间来回切换、根据结果调整方向、review diff 后再部署。新版桌面 App 就是为这种工作形态设计的。

核心更新：
- **侧边栏管理多会话**：并行运行多个 repo 的任务，按状态/项目/环境过滤；PR 合并或关闭后自动归档，侧边栏始终保持干净
- **集成终端**：测试和构建直接在 App 内运行，不需要切换窗口
- **内置文件编辑器**：打开文件、直接做小修改、保存，全部在 App 内完成
- **更快 diff 视图**：针对大文件改动重建
- **HTML/PDF 预览**：直接在 App 内打开，不需要起本地服务
- **拖拽布局**：终端、预览、diff、聊天任意组合，按你的工作方式排列
- **跨平台 SSH**：Mac 和 Linux 都支持远程机器连接

三种视图模式（Verbose / Normal / Summary）让你在"看到所有工具调用"和"只看最终结果"之间自由切换。新的快捷键（⌘+/）可以快速查看完整列表。

目前已对 Pro、Max、Team、Enterprise 计划用户开放，同时也通过 Claude API 提供。

## 播客

**Training Data**——From SEO to Agent-Led Growth: ProFound 的 James Cadwallader

AI 时代，营销的最大平台迁移正在发生：从"蓝链接搜索"转向"AI 概率生成回答"。但真正重要的变化不是互联网本身改变了，而是**谁在使用互联网**——从用蓝链接列表发现网站的消费者，变成了 AI agent。

**核心观点：** AI 正在替代消费者的角色。不是搜索入口变了，而是进门的那个人变了。消费者不再亲自翻网页，AI 替他访问所有相关网站、评估有用与否，然后把答案直接还给他。

**关键数据：** 一次淋浴头选购，ChatGPT 参考了 65 个网页来回答。人类有认知惰性，只看前几条结果；AI 有无限耐心和带宽，倾向使用互联网的长尾内容。这意味着营销人员需要重新理解自己的受众——现在的受众是一个有超级智能、无限带宽的 agent。

**关于"AI 内容 = 垃圾"的误解：** 有研究显示 50% 的网络内容已由 AI 生成。《纽约时报》做过盲测实验，53% 的读者更喜欢 AI 写的文章。"AI 不等于低质量，问题在于你要告诉 AI 一些它原本不知道的东西。"

**Agent-Led Growth 的本质：** 不是新渠道，是新用户。Claude Code 推荐 MongoDB 还是 Vercel？它从哪里获取这些信息？如果选定了 MongoDB，它如何处理互操作性？营销人员的新职责是：为超级智能配备关于你产品、品牌、服务的一切知识，让它能回答任何相关问题。

**"Dead Internet"的可能性：** 三内年内互联网可能变成主要被 AI 浏览的地方。人类不再点击网页，广告商业模式崩溃，内容创作激励消失。但社交媒体会成为人类最后的地盘——因为这里有人性，有真实想法交换，有金钱和地位激励。

**广告的未来：** 生成式广告在对话界面里将成为史上最高效的广告形式——超级个性化、实时匹配用户意图。但 B2B 场景里，agent 才是真正的购买决策者，人类希望它客观、不被左右。

**对营销人的建议：** ① 用工具（如 ProFound）理解你在不同 AI 平台上的展示情况，不能优化看不见的东西；② 营销团队必须用 agent，特别是 Agent-Led Growth，不接受这一点就等于不用互联网——只不过用的是传统电视和纸媒的思路。
